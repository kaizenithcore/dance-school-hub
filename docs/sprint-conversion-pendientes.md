# Sprint de Conversion - Pendientes por crear

## Objetivo
Registrar los elementos de conversion que aun no existen como funcionalidad publicada para implementarlos en siguientes sprints sin bloquear avances actuales.

## 1) Demo de producto

### 1.1 Video demo (pendiente)
Estado: pendiente de creacion.

Debe incluir:
- Duracion objetivo: 60 a 120 segundos.
- Guion base:
  - Problema operativo.
  - Flujo de matricula online.
  - Organizacion de horarios.
  - Renovaciones/lista de espera.
  - Cierre con CTA de prueba.
- Formatos: 16:9 para web y 9:16 para reutilizacion en redes.

Criterio de salida:
- Archivo final publicado y enlace incorporado en Hero y CTA.

### 1.2 Demo interactiva (pendiente)
Estado: pendiente de definicion tecnica.

Opciones sugeridas:
- Opcion A: recorrido guiado con datos mock dentro de la app.
- Opcion B: mini entorno sandbox read-only con un tenant demo.

Alcance minimo:
- Dashboard.
- Horarios.
- Matricula.
- Cobros.

Criterio de salida:
- URL publica accesible desde botones "Solicitar demo" o "Ver demo" segun estrategia comercial.

## 2) Guias publicas de onboarding
Estado: pendiente de publicacion.

Piezas a crear:
- Guia 1: Crear matricula en 5 minutos.
- Guia 2: Organizar horario de curso.
- Guia 3: Automatizar renovaciones.

Requisito:
- Cada guia debe tener URL publica (landing o blog/docs) para reemplazar etiquetas "Guia proximamente".

## 3) Prueba social y credibilidad
Estado: base implementada en landing; pendiente ampliar con nuevos casos reales.

Piezas a crear:
- Nuevos casos de testimonio (ademas de Escuela Danzante, Grinon).
- Material definitivo de foto/logo por escuela para reemplazar placeholders.
- KPI validable por cada caso (ejemplo: tiempo administrativo ahorrado, tasa de renovacion, reduccion de incidencias).
